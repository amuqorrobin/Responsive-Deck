# Contributing to Responsive Deck

First of all, thank you for your interest in Responsive Deck.

Responsive Deck is currently maintained under a **curated contribution model**.  
That means the repository is open source and visible to everyone, but code changes are merged selectively to keep the product focused, maintainable, and aligned with the roadmap.

## Contribution Policy

At this stage, we do **not** accept drive-by feature pull requests by default.

Please **open an Issue first** before spending time on a pull request, especially for:

- new features
- architectural changes
- UI/UX changes
- refactors
- dependency changes
- workflow/configuration updates

This helps avoid:

- feature bloat
- duplicated effort
- roadmap misalignment
- large PRs that are hard to review

## What is welcome

You are welcome to open an Issue for:

- bug reports
- security concerns
- compatibility issues
- small documentation fixes
- thoughtful feature proposals

## Pull Requests

Pull requests may be closed if they:

- were opened without prior discussion in an Issue
- introduce features not aligned with the roadmap
- make broad architectural changes without approval
- add maintenance burden disproportionate to the value

This is not meant to discourage contributors.  
It is meant to keep the project healthy and intentional.

## Recommended process

1. Open an Issue first
2. Describe the problem or proposal clearly
3. Wait for maintainer feedback
4. Only then begin implementation if the change is approved

## Development expectations

When a contribution is approved, please aim to:

- keep changes focused and minimal
- avoid unrelated refactors
- follow the existing project structure and style
- write clear commit messages
- explain the purpose of the change in the PR

## Code of collaboration

Please be respectful and constructive in Issues and pull requests.

## Maintainer note

Responsive Deck is currently developed with a strong product direction and a deliberate scope.  
For that reason, contribution review is intentionally strict.

Thank you for understanding and for taking the time to contribute thoughtfully.
